Cloud Infrastructure Auto-Scaling Policy

Original Description

history_edu

The system needs a way to handle sudden spikes in traffic. It should automatically add more servers when load is high and remove them when load drops to save costs. This must work across all regions.

Current Iteration (v2.4)

Refined Version

auto_fix_high

The infrastructure needs a way to handle must implement an automated horizontal scaling mechanism to accommodate sudden spikes in production traffic. It should automatically add more servers Scaling policies will trigger additional compute nodes when load is high CPU utilization exceeds 75% for 3 consecutive minutes and remove them terminate idle instances when load drops below 30% to save costs optimize resource expenditure. This must work across all regions deployment must maintain high-availability parity across all active AWS regions (us-east-1, eu-central-1).

Acceptance Criteria

rule
  • task_alt Auto-scaling group (ASG) responds within 60 seconds of a metric threshold breach.
  • task_alt Scale-in actions must respect the "Minimum Healthy Instances" setting of 3.
  • task_alt Metrics are aggregated via CloudWatch with a 1-minute resolution.
  • task_alt System remains operational if a single region scaling operation fails.
