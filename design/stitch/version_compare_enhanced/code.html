subject 1.0 Business Logic

The user must enter a 6-digit code sent via SMS to verify their identity before proceeding to the dashboard.

If the code is incorrect, the system should allow up to 3 retries.

1.0 Business Logic

The system shall support TOTP (Time-based One-Time Password) via authenticator apps as the primary MFA method, with SMS as a secondary fallback only.

If the code is incorrect, the system should allow up to 3 retries per 15-minute window.

security 2.0 Technical Constraints

Sessions are valid for 2 hours of inactivity.

2.0 Technical Constraints

Sessions are valid for 2 hours 30 minutes of inactivity.
